ER-диаграмма системы тюнинга автомобилей "TunningManual8080"

Описание системы

Система предназначена для подбора и оценки совместимости тюнинговых деталей для автомобилей. Каждая деталь оценивается по 10-балльной шкале совместимости с конкретной моделью автомобиля.

АВТОМОБИЛИ

Основная информация об автомобилях, для которых подбираются детали

  • Марка, модель, год выпуска

КАТЕГОРИИ_ДЕТАЛЕЙ

Классификация тюнинговых деталей

  • Название категории

ПРОИЗВОДИТЕЛИ

Информация о производителях деталей

  • Название производителя

ДЕТАЛИ

Каталог тюнинговых деталей

  • Наименование, цена
  • Связь с категорией и производителем

СОВМЕСТИМОСТЬ

Оценка совместимости деталей с автомобилями

  • Примечания по установке
  • Связь детали и автомобиля

КЛИЕНТЫ

Информация о клиентах сервиса

  • ФИО, телефон, email

ЗАКАЗЫ

Информация о заказах деталей

  • Дата, сумма заказа
  • Связь с клиентом и акцией

АКЦИИ

Маркетинговые акции и скидки

  • Название, описание, скидка

ОТЗЫВЫ

Отзывы клиентов о деталях

  • Оценка, текст отзыва
  • Дата отзыва

УСЛУГИ_УСТАНОВКИ

Услуги по установке деталей

  • Название, цена
  • Длительность в минутах

ЗАПИСИ_НА_УСЛУГИ

Записи клиентов на установку

  • Дата, время записи
  • Статус выполнения

АДМИНИСТРАТОРЫ

Администраторы системы

  • Логин, пароль

НАСТРОЙКИ_СИСТЕМЫ

Конфигурация системы

  • Ключ-значение настроек
  • Связь с администратором

Описание связей между сущностями:

📗 Связи 1 ко многим (One-to-Many)

  • КАТЕГОРИИ_ДЕТАЛЕЙ → ДЕТАЛИ: Одна категория содержит много деталей
  • ПРОИЗВОДИТЕЛИ → ДЕТАЛИ: Один производитель производит много деталей
  • АВТОМОБИЛИ → СОВМЕСТИМОСТЬ: Один автомобиль имеет много записей совместимости
  • ДЕТАЛИ → СОВМЕСТИМОСТЬ: Одна деталь совместима со многими автомобилями
  • КЛИЕНТЫ → ЗАКАЗЫ: Один клиент делает много заказов
  • АКЦИИ → ЗАКАЗЫ: Одна акция применяется к многим заказам
  • КЛИЕНТЫ → ОТЗЫВЫ: Один клиент оставляет много отзывов
  • ДЕТАЛИ → ОТЗЫВЫ: Одна деталь получает много отзывов
  • УСЛУГИ_УСТАНОВКИ → ЗАПИСИ_НА_УСЛУГИ: Одна услуга назначается много раз
  • КЛИЕНТЫ → ЗАПИСИ_НА_УСЛУГИ: Один клиент записывается много раз

📙 Особые типы связей

  • ЗАКАЗЫ ↔ ДЕТАЛИ: Многие ко многим (через связь содержит)
  • АДМИНИСТРАТОРЫ ↔ НАСТРОЙКИ_СИСТЕМЫ: Один к одному (управляет)

Основные возможности системы:

  • Подбор деталей: Оценка совместимости деталей с автомобилями
  • Каталог: Полный каталог тюнинговых деталей с фильтрацией
  • Заказы: Оформление заказов с учетом акций
  • Услуги: Запись на услуги установки
  • Отзывы: Система отзывов и рейтингов
  • Акции: Маркетинговые акции и скидки
  • Администрирование: Управление настройками системы
