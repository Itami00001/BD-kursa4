# Tunning Manual 8080 - Проект по базам данных

## Общая информация

**Название проекта:** Tunning Manual 8080  
**Тема:** Разработка базы данных для системы тюнинга автомобилей  
**Автор:** Студент  
**Учебный курс:** Базы данных, Лабораторные работы 8-14

## Цель проекта

Создание веб-приложения для подбора и оценки совместимости тюнинговых деталей для автомобилей с использованием современных технологий разработки программного обеспечения.

## Функциональные требования

### Основные функции:
- **Подбор деталей:** Оценка совместимости деталей с автомобилями по 10-балльной шкале
- **Каталог:** Полный каталог тюнинговых деталей с фильтрацией по категориям и производителям
- **Заказы:** Оформление заказов с учетом акций и скидок
- **Услуги:** Запись на услуги установки деталей
- **Отзывы:** Система отзывов и рейтингов деталей
- **Акции:** Управление маркетинговыми акциями
- **Администрирование:** Управление настройками системы

### Пользовательские роли:
- **Клиент:** Поиск деталей, оформление заказов, запись на услуги
- **Администратор:** Управление каталогом, заказами, настройками

## Технологический стек

### Backend:
- **Node.js** - среда выполнения JavaScript
- **Express.js** - веб-фреймворк
- **Sequelize** - ORM для работы с базой данных
- **PostgreSQL** - система управления базами данных

### DevOps:
- **Docker** - контейнеризация приложения
- **Docker Compose** - оркестрация контейнеров

### Документация:
- **Swagger** - автоматическая генерация документации API
- **Markdown** - техническая документация

## Архитектура приложения

### Многослойная архитектура:
1. **Presentation Layer** - REST API эндпоинты
2. **Business Logic Layer** - контроллеры и бизнес-правила
3. **Data Access Layer** - модели данных и ORM
4. **Database Layer** - PostgreSQL база данных

## Модель данных

### Основные сущности (14 таблиц):

1. **АВТОМОБИЛИ** - информация об автомобилях
2. **КАТЕГОРИИ_ДЕТАЛЕЙ** - классификация деталей
3. **ПРОИЗВОДИТЕЛИ** - производители деталей
4. **ДЕТАЛИ** - каталог тюнинговых деталей
5. **СОВМЕСТИМОСТЬ** - оценка совместимости деталей с автомобилями
6. **КЛИЕНТЫ** - информация о клиентах
7. **ЗАКАЗЫ** - заказы деталей
8. **АКЦИИ** - маркетинговые акции
9. **ОТЗЫВЫ** - отзывы клиентов о деталях
10. **УСЛУГИ_УСТАНОВКИ** - услуги по установке
11. **ЗАПИСИ_НА_УСЛУГИ** - записи на установку
12. **АДМИНИСТРАТОРЫ** - управление системой
13. **НАСТРОЙКИ_СИСТЕМЫ** - конфигурация

### Типы связей:
- **Один ко многим** - большинство связей
- **Многие ко многим** - заказы ↔ детали
- **Один к одному** - администраторы ↔ настройки

## API Эндпоинты

### Категории:
- `GET/POST /api/categories` - получить/создать категории
- `GET/PUT/DELETE /api/categories/:id` - управление категорией

### Детали:
- `GET/POST /api/parts` - получить/создать детали
- `GET/PUT/DELETE /api/parts/:id` - управление деталью
- `GET /api/parts/car/:carId/compatible` - совместимые детали

### Автомобили:
- `GET/POST /api/cars` - получить/создать автомобили
- `GET/PUT/DELETE /api/cars/:id` - управление автомобилем

### Производители:
- `GET/POST /api/manufacturers` - получить/создать производителей
- `GET/PUT/DELETE /api/manufacturers/:id` - управление производителем

### Клиенты:
- `GET/POST /api/customers` - получить/создать клиентов
- `GET/PUT/DELETE /api/customers/:id` - управление клиентом
