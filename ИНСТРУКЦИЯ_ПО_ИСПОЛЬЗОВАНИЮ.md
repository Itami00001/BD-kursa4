# Инструкция по использованию системы "Tunning Manual 8080"

## Обзор

Система "Tunning Manual 8080" предназначена для подбора и оценки совместимости тюнинговых деталей для автомобилей. Проект разработан с использованием Node.js, Express, PostgreSQL и Docker.

## Установка и запуск

### 1. Настройка переменных окружения

Скопируйте файл `.env.sample` в `.env`:
```bash
cp .env.sample .env
```

Отредактируйте `.env` при необходимости:
```env
DB_HOST=localhost
DB_USER=postgres
DB_PASSWORD=1
DB_NAME=tunning_manual_db
DB_PORT=5432
NODE_DOCKER_PORT=8080
```

### 2. Запуск с Docker

Запустите все сервисы:
```bash
docker-compose up -d --build
```

Проверьте статус контейнеров:
```bash
docker-compose ps
```

Просмотрите логи:
```bash
docker-compose logs -f app
```

### 3. Остановка

Остановите все контейнеры:
```bash
docker-compose down
```

## API Endpoints

|-------|-----|----------|
| GET | `/api/categories` | Получить все категории |
| POST | `/api/categories` | Создать новую категорию |
| GET | `/api/categories/:id` | Получить категорию по ID |

## Разработка

### Структура кода

1. **Models (Модели)** - Определяют структуру таблиц базы данных и связи между ними
2. **Controllers (Контроллеры)** - Содержат бизнес-логику для CRUD операций
3. **Routes (Маршруты)** - Определяют эндпоинты API и связывают их с контроллерами
4. **Database Config** - Конфигурация подключения к базе данных

### Добавление новой сущности

1. Создайте модель в `app/models/`
2. Добавьте модель в `app/models/index.js`
3. Создайте контроллер в `app/controllers/`
4. Создайте маршруты в `app/routes/`
5. Подключите маршруты в `server.js`

### Пример добавления новой сущности "Reviews"

```javascript
// 1. Модель app/models/review.model.js
module.exports = (sequelize, Sequelize) => {
  const Review = sequelize.define("reviews", {
    rating: { type: Sequelize.INTEGER, allowNull: false },
    text: { type: Sequelize.TEXT }
  });
  return Review;
};

// 2. Добавить в app/models/index.js
db.reviews = require("./review.model.js")(sequelize, Sequelize);

// 3. Контроллер app/controllers/review.controller.js
exports.create = (req, res) => { /* логика */ };
exports.findAll = (req, res) => { /* логика */ };
// ... другие методы

// 4. Маршруты app/routes/review.routes.js
module.exports = app => {
  const reviews = require("../controllers/review.controller.js");
  var router = require("express").Router();
  router.post("/", reviews.create);
  router.get("/", reviews.findAll);
  app.use('/api/reviews', router);
};

// 5. Подключить в server.js
require("./app/routes/review.routes")(app);
```

## Возможные проблемы

### 1. Проблемы с подключением к БД

**Ошибка:** `ECONNREFUSED`

**Решение:**
- Убедитесь что Docker запущен
- Проверьте что контейнер postgresdb запущен: `docker-compose ps`
- Проверьте переменные окружения в `.env`

### 2. Ошибка синхронизации БД

**Ошибка:** `Failed to sync db`

**Решение:**
- Проверьте правильность определений моделей
- Убедитесь что все связи определены корректно
- Перезапустите контейнеры: `docker-compose down && docker-compose up -d`

### 3. Порт уже занят

**Ошибка:** `Port 8080 is already in use`

**Решение:**
- Измените порт в `.env`: `NODE_DOCKER_PORT=8081`
- Или остановите процесс использующий порт: `netstat -ano | findstr :8080`
